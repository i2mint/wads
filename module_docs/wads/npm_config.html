wads.npm_config

Read NPM CI configuration from a package.json wads.ci block.

This is the NPM-side analog of wads.ci_config.CIConfig (which reads [tool.wads.ci] from pyproject.toml). npm ignores unknown top-level keys, so wads namespaces its config under a top-level "wads" key:

{
  "name": "@scope/my-widget",
  "wads": {
    "ci": {
      "subdir": "js",
      "nodeVersions": ["20", "22", "24"],
      "publish": {"enabled": false, "marker": "[publish-npm]"}
    }
  }
}

The reusable workflow (i2mint/wads/.github/workflows/npm-ci.yml) parses the same block in CI via node; this class is the parsing/validation surface used from Python (tests, populate validation, inspection tooling).

wads.npm_config.NPM_CI_DEFAULTS = {'buildCommand': 'npm run build --if-present', 'lintCommand': 'npm run lint --if-present', 'nodeVersions': ['20', '22', '24'], 'publishNode': '24', 'subdir': 'js', 'testCommand': 'npm test --if-present'}

Defaults applied when a field is absent from wads.ci.

wads.npm_config.NPM_PUBLISH_DEFAULTS = {'access': 'public', 'enabled': False, 'marker': '[publish-npm]', 'provenance': True, 'registry': 'https://registry.npmjs.org', 'trustedPublishing': True}

Defaults for the nested wads.ci.publish block.
